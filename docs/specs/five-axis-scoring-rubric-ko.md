# 5축 라켓 스코어링 기준서

작성일: 2026-03-17
최종 수정일: 2026-07-24
문서 목적: 사용자에게 노출되는 5개 해석 점수를 AI가 일관되게 계산하고 설명할 수 있도록 세부 기준을 정의한다.

## 1. 범위

이 문서는 아래 5개 공개 점수에만 적용한다.

- 파워
- 컨트롤
- 스핀
- 편안함
- 안정성

내부 보조 신호인 조작성, 관용성, 팔 부담 리스크 등은 이 문서의 직접 노출 범위 밖이다.

## 2. 점수 구조

### 내부 계산 점수

- 범위: `0~100`
- 용도: 추천, 정렬, 설명 생성, 비교

### 사용자 노출 점수

- 축 범위: 각 축 정수 `0~5`
- 총점 범위: 다섯 축의 합인 정수 `10~15`
- 표시: 축 `N/5`, 총점 `N/15`, 소수점 없음
- 용도: 라켓 카드, 상세, 비교, 추천 결과의 공개 설명

내부 raw 점수는 공개 정수로 덮어쓰지 않는다. 정렬과 추천은 `0~100` raw를 계속 사용한다.

### 계산 순서

1. 라켓 스펙 정규화
2. 세그먼트 분류
3. 5축 내부 점수 계산
4. raw v3 5축 합으로 목표 총점 계산
5. bounded projection과 deterministic largest-remainder로 공개 축 정수 배정
6. 공개 축 합과 목표 총점 불변식 확인
7. 설명 문장 생성

## 3. 세그먼트 규칙

공개 점수는 반드시 같은 세그먼트 안에서 계산한다.

### 1차 세그먼트

- `light_beginner`: 260~285g
- `balanced_mid`: 286~300g
- `performance`: 301~315g
- `tour`: 316g 이상

### 2차 세그먼트

- `98_class`: 97~98 sq.in
- `100_class`: 99~100 sq.in
- `oversize_class`: 101 sq.in 이상

### 예외 규칙

- 헤드사이즈 105 이상은 별도 풀에서 본다.
- 주니어 전환형과 성인 퍼포먼스형은 직접 비교하지 않는다.

## 4. 입력 데이터 우선순위

### 필수 입력값

- `weight_grams_unstrung`
- `balance_mm_unstrung`
- `head_size_sq_in`
- `beam_mm_min`
- `beam_mm_mid`
- `beam_mm_max`
- `string_pattern_mains`
- `string_pattern_crosses`
- `stiffness_ra`
- `swing_weight`

### 출처 우선순위

1. 브랜드 공식 스펙
2. 한국 공식 유통사
3. 전문 판매처
4. 그 외 판매처

## 5. 파생 변수 정의

- `beam_avg_mm = (beam_mm_min + beam_mm_mid + beam_mm_max) / 3`
- `head_light_index = 343 - balance_mm_unstrung`
- `pattern_open_index = head_size_sq_in / (string_pattern_mains * string_pattern_crosses)`
- `pattern_dense_index = 1 / pattern_open_index`
- `easy_depth_proxy = normalized(head_size_sq_in) + normalized(beam_avg_mm) + normalized(swing_weight)`

모든 파생 변수는 세그먼트 내에서 표준화해서 사용한다.

## 6. 축별 계산 기준

## 6-1. 파워

### 핵심 의미

적은 힘으로도 공이 깊고 길게 뻗는 성향.

### 주 가중치

- 스윙웨이트: 35%
- 강성 RA: 25%
- 빔 평균 두께: 20%
- 헤드사이즈: 20%

### 감점 요인

- 지나치게 낮은 스윙웨이트
- 매우 얇은 빔

### 해석 기준

- `5/5`: 같은 세그먼트에서 깊은 볼을 만들기 쉬운 성향이 매우 강함
- `2/5~3/5`: 다른 축과 균형을 이루는 중간 성향
- `0/5~1/5`: 파워가 상대적 강점이 아니며 사용자의 힘이 더 필요할 수 있음

## 6-2. 컨트롤

### 핵심 의미

탄도와 방향을 예측하고 코스를 제어하기 쉬운 성향.

### 주 가중치

- 작은 헤드사이즈: 30%
- 촘촘한 패턴: 25%
- 얇은 빔: 20%
- 파워 역보정: 25%

### 감점 요인

- 과도한 프레임 파워
- 지나치게 큰 헤드사이즈

### 해석 기준

- `5/5`: 강하게 쳐도 방향과 탄도 제어가 쉬운 성향이 매우 강함
- `2/5~3/5`: 다른 축과 균형을 이루는 중간 성향
- `0/5~1/5`: 컨트롤이 상대적 강점이 아니며 프레임 파워 관리가 더 필요할 수 있음

## 6-3. 스핀

### 핵심 의미

회전량을 만들기 쉬운 프레임 성향.

### 주 가중치

- 오픈 패턴 지수: 40%
- 헤드사이즈: 20%
- 조작성 보조 신호: 20%
- 스핀 프레임 태그 보조: 20%

### 감점 요인

- 촘촘한 패턴
- 지나치게 낮은 조작성

### 해석 기준

- `5/5`: 탑스핀과 킥 계열에서 회전 형성이 매우 쉬운 성향
- `2/5~3/5`: 다른 축과 균형을 이루는 중간 성향
- `0/5~1/5`: 스핀보다 직선적 타구가 더 자연스러운 성향

## 6-4. 편안함

### 핵심 의미

타구 시 충격을 부드럽게 받아내는 성향.

### 주 가중치

- 낮은 강성 RA: 40%
- 부드러운 빔 영역: 20%
- 안정성: 20%
- 소프트 스트링 친화성: 20%

### 감점 요인

- 매우 높은 강성
- 지나치게 가볍고 흔들리는 프레임

### 해석 기준

- `5/5`: 팔 부담 완화 관점에서 편안함 성향이 매우 강함
- `2/5~3/5`: 다른 축과 균형을 이루는 중간 성향
- `0/5~1/5`: 충격이 비교적 직접적으로 전달될 수 있는 성향

## 6-5. 안정성

### 핵심 의미

임팩트 순간 버텨주는 힘과 비틀림 저항.

### 주 가중치

- 스윙웨이트: 45%
- 정적 무게: 35%
- 빔 평균 두께: 20%

### 감점 요인

- 너무 낮은 스윙웨이트
- 지나치게 가벼운 프레임

### 해석 기준

- `5/5`: 강한 볼 대응과 블로킹에서 흔들림이 매우 적은 성향
- `2/5~3/5`: 다른 축과 균형을 이루는 중간 성향
- `0/5~1/5`: 강한 임팩트에서 버티는 힘이 상대적 강점이 아닌 성향

## 7. 공개 점수 변환 규칙

공개 점수는 각 축을 독립 반올림하지 않는다. 다섯 축과 총점이 동시에 일관되도록 아래 순서로 결정한다.

1. canonical 공개 축 순서를 `파워, 컨트롤, 스핀, 편안함, 안정성`으로 고정한다.
2. 각 raw v3 값을 `0~100`으로 제한하고 다섯 값의 합을 `rawSum`으로 둔다.
3. 목표 총점 `T = round(10 + rawSum / 100)`을 계산한다. `T`는 항상 정수 `10~15`다.
4. 각 축의 초기 공개값을 `raw / 20`으로 두고 `0~5` 경계 안에서 합이 `T`가 되도록 bounded simplex projection을 수행한다.
5. 투영값의 정수부를 먼저 배정하고 남은 점수는 largest-remainder 순서로 한 점씩 배분한다.
6. remainder가 같으면 raw 점수가 큰 축을 먼저, raw도 같으면 canonical 축 순서를 먼저 배정한다.
7. 결과의 각 축이 정수 `0~5`이고 다섯 축 합이 정확히 `T`인지 확인한다.

예시:

- raw `[100, 60, 50, 40, 0]` → 공개 `[5, 3, 3, 2, 0]` → 총 `13/15`
- raw `[50, 50, 50, 50, 50]` → 공개 `[3, 3, 3, 2, 2]` → 총 `13/15`

UI에는 축 `N/5`, 총점 `N/15`만 표시하고 소수점을 노출하지 않는다. 정렬·추천·유사도 계산에는 공개 정수가 아니라 raw v3를 사용한다.

## 8. 점수 보류 규칙

아래 조건에서는 점수를 그대로 노출하지 않거나 신뢰도를 낮춘다.

- 필수 입력값 2개 이상 누락
- `swing_weight`와 `stiffness_ra`가 모두 누락
- 공식 출처와 판매처 스펙이 크게 충돌
- 동일 모델의 연식 구분이 불명확

### 처리 방식

- `confidence_score < 0.6`: 점수 노출은 가능하되 "데이터 확인 중" 배지
- `confidence_score < 0.4`: 추천 후보에서 제외하거나 하단 노출

## 9. 설명 문장 생성 규칙

### 한 줄 총평

구조

- 높은 점수 2개
- 사용자 세그먼트 1개
- 낮은 점수 1개는 주의 문장으로 분리

예시

- "쉬운 파워와 안정성을 앞세운 중급 성장형 라켓"
- "스핀과 컨트롤이 살아 있는 빠른 스윙형 라켓"

### 강점 문장

예시 템플릿

- 파워 `4/5` 이상: "적은 힘으로도 깊은 볼을 만들기 쉬운 성향입니다."
- 컨트롤 `4/5` 이상: "강하게 쳐도 방향과 탄도 관리가 쉬운 편입니다."
- 스핀 `4/5` 이상: "탑스핀을 걸 때 라켓의 도움을 받기 좋은 편입니다."
- 편안함 `4/5` 이상: "타구 충격이 비교적 부드럽게 전달되는 편입니다."
- 안정성 `4/5` 이상: "강한 볼을 받아낼 때 라켓이 쉽게 밀리지 않는 편입니다."

### 주의 문장

예시 템플릿

- 파워 `1/5` 이하: "스윙이 작으면 볼 길이가 부족하게 느껴질 수 있습니다."
- 컨트롤 `1/5` 이하: "타점이 밀리면 볼이 길어질 수 있습니다."
- 편안함 `1/5` 이하: "민감한 팔에는 단단하게 느껴질 수 있습니다."
- 안정성 `1/5` 이하: "강한 임팩트에서는 다소 가볍게 느껴질 수 있습니다."

## 10. 카드 노출 규칙

### 리스트 카드

- 5축 중 상위 2개와 평균 이하 1개만 선택 표시하지 않는다.
- 리스트에서는 강점 2개만 노출한다.
- 부정 점수는 리스트에서 과도하게 강조하지 않는다.

### 상세 페이지

- 5축 전체 노출
- 각 축 `N/5`와 총점 `N/15` 노출
- 5축 각각 짧은 설명 제공
- 주의 포인트 1~2개 분리 노출

### 비교 페이지

- 5축 전체를 동일 스케일로 노출
- 가장 큰 차이가 나는 축을 자동 하이라이트

## 11. 액세서리 추천 연결 규칙

### 스트링

- 파워 `4/5` 이상이고 컨트롤 `2/5` 이하: 컨트롤형 폴리 우선
- 스핀 `4/5` 이상: 스핀형 폴리 우선
- 편안함 `2/5` 이하: 멀티 또는 소프트 폴리 우선

### 그립

- 편안함 낮음: 쿠션형 우선
- 안정성 낮음: 두께감 있는 세팅 후보 제공
- 손땀 조건은 사용자 입력값과 결합

## 12. 금지 규칙

- 서로 유사한 뜻의 점수를 동시에 노출하지 않는다.
- 점수의 절대 우열처럼 보이게 말하지 않는다.
- 스펙 누락 상태에서 단정형 문장을 쓰지 않는다.
- 브랜드 홍보 문구를 AI 해석처럼 보이게 쓰지 않는다.

## 13. MVP 운영 체크리스트

- 5축 필수 입력값 충족 여부
- 세그먼트 분류 정상 여부
- 5축 점수 계산 완료 여부
- 공개 5축이 각각 정수 `0~5`인지 확인
- 공개 5축 합이 `round(10 + rawSum / 100)` 및 표시 총점과 같은지 확인
- UI가 축 `N/5`, 총 `N/15`를 소수점 없이 표시하는지 확인
- `confidence_score` 계산 여부
- 한 줄 총평 생성 여부
- 주의 포인트 생성 여부
- 추천 스트링 2개 이상 연결 여부
- 추천 그립 1개 이상 연결 여부
