# AI 라켓 스코어링 방법론

작성일: 2026-03-17
최종 수정일: 2026-07-24
문서 목적: 라켓 DB를 기반으로 AI가 객관적으로 라켓을 분류, 점수화, 설명, 추천할 수 있는 기준점을 정의한다.

## 1. 핵심 원칙

이 플랫폼의 라켓 해석은 사람의 감상문이 아니라 데이터 기반 AI 해석이어야 한다.

기본 원칙은 아래와 같다.

1. 브랜드 마케팅 문구보다 정규화된 스펙을 우선한다.
2. 모든 점수는 명시된 계산 규칙을 따른다.
3. 같은 세그먼트 안에서 비교한다.
4. 설명 문장은 점수와 스펙에서 생성한다.
5. 사람이 개입하는 영역은 예외 처리와 QA로 제한한다.

## 2. AI가 판단할 수 있으려면 필요한 DB

라켓별 최소 필수 필드는 아래와 같다.

- 브랜드
- 시리즈
- 모델명
- 출시연도
- 무게
- 밸런스
- 헤드사이즈
- 길이
- 빔 두께
- 스트링 패턴
- 강성 RA
- 스윙웨이트
- 국내 판매 여부
- 국내 실판매가

보조 필드

- 그립 사이즈
- 권장 텐션
- 프레임 설명 문구
- 이미지 품질 메타데이터

## 3. 세그먼트 분류 기준

AI는 먼저 라켓을 같은 부류끼리 묶는다.

### 무게 세그먼트

- `light_beginner`: 260~285g
- `balanced_mid`: 286~300g
- `performance`: 301~315g
- `tour`: 316g 이상

### 헤드 세그먼트

- `midplus_98`: 97~98 sq.in
- `midplus_100`: 99~100 sq.in
- `oversize`: 101 이상

### 사용자 목적 세그먼트

- `beginner`
- `improver`
- `competitive`

라켓의 공개 점수는 전체 풀에서 계산하지 않고 위 세그먼트 내 상대값으로 계산한다.

## 4. 내부 표준화 규칙

AI가 일관되게 판단하려면 입력값을 먼저 정규화해야 한다.

### 파생 변수

- `beam_avg_mm = (beam_mm_min + beam_mm_mid + beam_mm_max) / 3`
- `pattern_open_index = head_size_sq_in / (string_pattern_mains * string_pattern_crosses)`
- `balance_head_light_index = 343 - balance_mm_unstrung`
- `price_band = budget bucket`

### 누락값 처리

- `swing_weight`가 없으면 같은 시리즈/유사 스펙의 평균으로 임시 대체
- `stiffness_ra`가 없으면 공식 스펙 우선 재수집
- 누락이 큰 상품은 사용자 노출 시 `confidence_score`를 낮춘다

## 5. AI 해석 축

공개 노출 축은 아래 5개로 고정한다.

- 파워
- 컨트롤
- 스핀
- 편안함
- 안정성

내부적으로는 필요시 아래 보조 축을 더 계산할 수 있다.

- 조작성
- 관용성
- 파워 효율
- 발리 적합도
- 복식 적합도
- 입문 난이도
- 팔 부담 리스크

## 6. 내부 점수 계산식 초안

모든 내부 점수는 `0~100`으로 계산한다.

점수는 우선 각 변수의 세그먼트 내 표준화값을 만든 뒤 가중합으로 계산한다.

### 6-1. 파워

영향도가 큰 변수

- 스윙웨이트: 높을수록 가산
- 강성 RA: 높을수록 가산
- 빔 평균 두께: 두꺼울수록 가산
- 헤드사이즈: 클수록 가산

초안 식

`penetration_raw = 0.35*SW + 0.25*RA + 0.20*BEAM + 0.20*HEAD`

### 6-2. 컨트롤

영향도가 큰 변수

- 헤드사이즈: 작을수록 가산
- 파워 과다: 감점
- 패턴 밀도: 촘촘할수록 가산
- 빔 평균: 얇을수록 가산

초안 식

`control_raw = 0.30*(-HEAD) + 0.25*PATTERN_DENSE + 0.20*(-BEAM) + 0.25*(-PENETRATION)`

### 6-3. 스핀

영향도가 큰 변수

- 오픈 패턴 지수: 높을수록 가산
- 헤드사이즈: 클수록 소폭 가산
- 조작성: 높을수록 가산
- 프레임 스핀 성향 태그: 보조 가산

초안 식

`spin_raw = 0.40*PATTERN_OPEN + 0.20*HEAD + 0.20*MANEUVER + 0.20*SPIN_TAG`

### 6-4. 편안함

영향도가 큰 변수

- 강성 RA: 낮을수록 가산
- 빔 평균: 얇거나 중간이면 가산
- 무게와 안정성: 지나치게 가볍고 불안정하면 감점
- 권장 스트링 친화성: 멀티/소프트폴리 친화 시 가산

초안 식

`comfort_raw = 0.40*(-RA) + 0.20*BEAM_SOFT_ZONE + 0.20*STABILITY + 0.20*STRING_SOFT_FRIENDLY`

### 6-5. 안정성

영향도가 큰 변수

- 스윙웨이트: 높을수록 가산
- 정적 무게: 높을수록 가산
- 빔 평균: 너무 얇지 않으면 가산

초안 식

`stability_raw = 0.45*SW + 0.35*WEIGHT + 0.20*BEAM`

### 6-6. 조작성

영향도가 큰 변수

- 정적 무게: 낮을수록 가산
- 스윙웨이트: 낮을수록 가산
- 헤드 라이트 성향: 높을수록 가산

초안 식

`maneuver_raw = 0.35*(-WEIGHT) + 0.35*(-SW) + 0.30*HEAD_LIGHT`

### 6-7. 관용성

영향도가 큰 변수

- 헤드사이즈: 클수록 가산
- 안정성: 높을수록 가산
- 파워/스위트존 넓이 추정치: 중간 이상이면 가산

초안 식

`forgiveness_raw = 0.45*HEAD + 0.30*STABILITY + 0.25*EASY_DEPTH`

## 7. 공개 점수 변환 규칙

사용자에게는 축별 정수 `0~5`와 다섯 축 합계인 정수 `10~15`를 보여준다. 내부 raw v3 `0~100`은 그대로 보존해 정렬과 추천에 사용한다.

변환 방식

1. canonical 순서의 raw 5축을 각각 `0~100`으로 제한한다.
2. `rawSum`을 구하고 목표 총점 `T = round(10 + rawSum / 100)`을 계산한다.
3. 각 축의 초기 공개값 `raw / 20`을 `0~5` 경계 안에서 합이 `T`가 되도록 bounded simplex projection한다.
4. 투영값의 정수부를 배정하고 남은 점수는 deterministic largest-remainder로 배분한다.
5. remainder 동률은 raw 내림차순, 이후 파워·컨트롤·스핀·편안함·안정성 순으로 해소한다.
6. 각 공개 축이 정수 `0~5`이고 합이 정확히 `T`인지 검증한다.

표시 규칙

- 축: `N/5`
- 총점: `N/15`
- 소수점: 사용하지 않음
- 정렬·추천·유사도: 공개 정수가 아니라 raw v3 사용

공개 UI에는 아래 5개만 노출한다.

- `public_score_penetration`
- `public_score_control`
- `public_score_spin`
- `public_score_comfort`
- `public_score_stability`

조작성과 관용성은 추천 로직용 내부 신호로 유지할 수 있지만, MVP에서는 사용자에게 직접 보여주지 않는다.

## 8. 객관성 확보 장치

AI가 마음대로 판단하는 구조가 아니라 아래 장치를 둔다.

### 1. 소스 우선순위

- 공식 스펙
- 공식 유통사
- 전문 판매처
- 기타 판매처

### 2. 데이터 신뢰도 점수

각 라켓마다 `confidence_score`를 둔다.

반영 요소

- 핵심 필드 완성도
- 공식 소스 비중
- 상충 데이터 여부
- 스윙웨이트/RA 확보 여부

### 3. 설명 추적 가능성

모든 공개 설명은 아래 구조로 생성한다.

- 점수가 높은 축 2개
- 점수가 낮은 축 1개
- 핵심 스펙 2~3개
- 추천 사용자 유형

즉, 설명은 문장 생성이 아니라 구조화된 근거에서 나온다.

## 9. AI 설명 생성 규칙

### 한 줄 총평 생성

예시 템플릿

- "쉬운 파워와 넓은 관용성을 앞세운 입문-중급용 라켓"
- "빠른 스윙에서 컨트롤과 스핀을 살리기 좋은 퍼포먼스형 라켓"

### 강점 설명 생성

예시

- 파워 `5/5`: 스윙웨이트와 프레임 파워가 높아 적은 힘으로도 깊은 볼을 만들기 쉽습니다.

### 주의점 생성

예시

- 컨트롤이 평균보다 낮아 타점이 밀리면 볼이 길어질 수 있습니다.
- 조작성이 낮아 복식 전위에서는 무겁게 느껴질 수 있습니다.

## 10. 스트링 추천 AI 규칙

라켓 스코어와 사용자 성향을 함께 반영한다.

### 입력

- 라켓 7축 점수
- 팔 부담 여부
- 실력 수준
- 스핀 선호
- 선호 타구감

### 출력

- 추천 스트링 타입
- 추천 장력 범위
- 추천 이유

### 예시 규칙

- 파워 `4/5` 이상, 컨트롤 `2/5` 이하
  - `control_poly`, `hybrid_control`
- 편안함 `2/5` 이하, 팔 부담 있음
  - `multi_soft`, `soft_poly_low_tension`
- 스핀 `3/5` 이상, 중급 이상
  - `spin_poly`

## 11. 그립 추천 AI 규칙

### 입력

- 편안함 점수
- 손땀 성향
- 계절/환경
- 타구감 선호

### 출력

- 리플레이스먼트 그립 프로필
- 오버그립 프로필
- 추천 이유

### 예시 규칙

- 편안함 낮음
  - `cushioned_replacement`
- 손땀 많음
  - `dry_overgrip`
- 타구감 선명도 선호
  - `thin_overgrip`

## 12. 사람의 역할

사용자가 말한 방향대로 이 시스템의 중심은 AI다.

사람이 맡는 영역은 아래로 최소화한다.

- 공식 스펙 출처 확인
- 중복 모델 병합 승인
- 상충 데이터 예외 처리
- 이미지 라이선스 확인

즉, 사람은 "판단자"가 아니라 "데이터 QA 관리자" 역할에 가깝다.

## 13. MVP 구현 우선순위

1. 라켓 DB 필드 완성
2. 세그먼트 정의
3. 5축 공개 점수와 보조 신호 계산식 구현
4. 공개 축 정수 `0~5`와 총점 `10~15`의 bounded projection 및 largest-remainder 변환
5. 설명 문장 생성 규칙 구현
6. 스트링/그립 추천 규칙 구현
7. confidence_score 노출

## 14. 최종 결론

이 플랫폼의 경쟁력은 AI가 라켓을 일관된 기준으로 해석하는 데서 나온다.

즉, 우리가 만들어야 하는 것은 단순 카탈로그가 아니라 아래 엔진이다.

- 한국 판매 라켓 DB 엔진
- AI 기능성 점수 엔진
- AI 설명 생성 엔진
- AI 액세서리 추천 엔진

이 네 가지가 합쳐져야 사용자가 "여긴 진짜 라켓을 읽어준다"라고 느낄 수 있다.
