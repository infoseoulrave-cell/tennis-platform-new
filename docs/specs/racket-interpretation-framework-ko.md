# 라켓 해석 프레임워크 초안

작성일: 2026-03-17
최종 수정일: 2026-07-24
문서 목적: 라켓의 기능성을 우리만의 언어로 해석하고, 추천 스트링·그립·사진 노출 기준까지 하나의 체계로 정의한다.

## 1. 왜 이 프레임워크가 중요한가

이 플랫폼의 본질은 단순 상품 나열이 아니라 "라켓을 읽어주는 능력"에 있다.

사용자는 스펙 표보다 아래 질문에 대한 답을 원한다.

- 이 라켓은 공이 쭉 뻗는 편인가
- 방향 제어가 쉬운가
- 팔에 부담이 적은가
- 스핀을 걸기 쉬운가
- 나 같은 스타일에 맞는가

따라서 브랜드 문구를 반복하는 대신, 플랫폼이 직접 해석한 공통 기준이 필요하다.

## 2. 워킹 콘셉트

서비스 내 라켓 해석 언어의 워킹 타이틀은 `Racket DNA`로 둔다.

사용자에게는 아래 두 층으로 보여준다.

1. 한눈에 보는 핵심 강점
2. 세부 기능성 점수 카드

예시

- 파워 `4/5`
- 컨트롤 `2/5`
- 스핀 `3/5`
- 편안함 `2/5`
- 안정성 `2/5`
- 총점 `13/15`

주의 포인트는 별도 노출한다.

- 빠른 스윙이 필요함
- 타점이 늦으면 길게 날 수 있음
- 손목 부담이 있을 수 있음

## 3. 점수 체계 원칙

### 사용자 노출 점수

- 축 범위: 각각 정수 `0~5`
- 총점 범위: 다섯 축 합계인 정수 `10~15`
- 표시: 축 `N/5`, 총점 `N/15`, 소수점 없음
- 기준: 같은 세그먼트의 raw v3를 바탕으로 한 bounded projection
- 목적: 직관적으로 "이 라켓의 성향"을 읽게 해줌

### 내부 운영 점수

- 범위: `0` ~ `100`
- 기준: 스펙 기반 계산 + 운영 검수 + 시타/리뷰 보정
- 목적: 추천 로직, 정렬, 필터링, A/B 테스트에 활용

내부 raw v3 `0~100`은 정렬과 추천에 그대로 사용한다. 공개값은 목표 총점 `round(10 + rawSum / 100)`을 구한 뒤 bounded projection과 deterministic largest-remainder로 배정하며, 다섯 공개 축의 정수 합은 목표 총점과 정확히 같아야 한다.

## 4. 세그먼트 기준

모든 라켓을 한 줄로 세우면 점수가 왜곡된다. 따라서 아래 세그먼트 안에서 상대평가한다.

- 라이트 입문형: 260~285g
- 밸런스형: 286~300g
- 퍼포먼스형: 301~315g
- 투어형: 316g 이상
- 오버사이즈형: 헤드사이즈 105 이상 별도 관리
- 주니어 전환형: 성인 입문 전환 라켓 별도 관리

이렇게 해야 280g 입문 라켓과 315g 투어 라켓을 같은 기준으로 비교하는 오류를 피할 수 있다.

## 5. 핵심 해석 축

사용자에게 노출하는 기본 축은 5개로 제한하는 것이 적절하다.

이유

- 한눈에 이해하기 쉽다.
- 라켓별 차이가 과하게 분산되지 않는다.
- 초중급 사용자도 비교가 가능하다.
- 해석 축이 많아질수록 중복과 혼란이 커진다.

### 1. 파워

- 정의: 공이 전진성 있게 뻗는 느낌과 쉬운 파워 전달
- 높을수록: 깊은 볼을 쉽게 만들기 좋음
- 낮을수록: 사용자가 직접 힘을 더 내야 함

### 2. 컨트롤

- 정의: 방향성과 탄도 예측 가능성
- 높을수록: 라인 공략, 코스 제어에 유리
- 낮을수록: 파워는 있으나 오차가 커질 수 있음

### 3. 스핀

- 정의: 회전량을 만들기 쉬운 성향
- 높을수록: 탑스핀/킥서브에 유리

### 4. 편안함

- 정의: 타구 충격을 부드럽게 흡수하는 정도
- 높을수록: 팔꿈치/손목 부담 완화에 유리

### 5. 안정성

- 정의: 임팩트 순간 비틀림 저항과 버티는 힘
- 높을수록: 강한 볼 대응, 블로킹, 리턴에서 유리

주의

- 조작성과 관용성은 유용한 개념이지만, 사용자 노출 축으로는 우선 숨기는 편이 좋다.
- 이 값들은 내부 추천용 보조 신호로 유지할 수 있다.

## 6. 점수 산정 로직 초안

이 문서의 점수 축은 별도 문서 [AI 라켓 스코어링 방법론](/Users/minho/Documents/프로젝트/tennis-platform/docs/specs/ai-racket-scoring-methodology-ko.md)의 계산 기준을 따른다.

### 1단계. 스펙 기반 기본값 산정

주요 입력값

- 무게
- 밸런스
- 헤드사이즈
- 빔 두께
- 스트링 패턴
- 강성
- 스윙웨이트

예시 방향

- 헤드사이즈와 빔이 크면 파워/관용성 가산
- RA가 낮고 멀티 친화적이면 편안함 가산
- 98 헤드, 얇은 빔, 촘촘한 패턴이면 컨트롤 가산
- 헤드 라이트와 낮은 스윙웨이트는 조작성 가산

### 2단계. AI 기반 보정

브랜드 카피보다 정규화된 스펙과 세그먼트 비교값을 우선하며, AI가 공개 점수와 설명 문장을 생성한다.

- 세그먼트 내 상대 위치 반영
- 국내 판매 모델 기준 반영
- 누락값과 신뢰도 점수 반영

### 3단계. 예외 QA

사람이 개입하는 영역은 최소화하고 예외 처리 중심으로 둔다.

- 상충 스펙 정리
- 중복 모델 병합
- 설명 오류 검수
- 이미지 라이선스 검수

## 7. 사용자 화면에서의 표현 방식

### 라켓 카드

- 대표 이미지
- 시리즈/모델명
- 핵심 3개 점수만 노출
- 예산대
- 추천 대상 태그

예시

- 파워 `4/5`
- 편안함 `3/5`
- 안정성 `2/5`

### 라켓 상세 페이지

- 히어로 이미지
- 기능성 점수 카드 5축
- 한 줄 총평
- 추천 플레이어
- 주의 포인트
- 추천 스트링
- 추천 그립
- 판매처 링크

### 비교 페이지

- 2~4개 라켓의 5축 비교
- 어떤 사용자에게 더 맞는지 문장형 요약

## 8. 점수 설명 문장 템플릿

점수는 숫자만 보여주면 부족하다. 모든 라켓에는 짧은 설명 문장을 붙인다.

예시

- 파워 `5/5`: 적은 힘으로도 깊고 길게 뻗는 볼을 만들기 쉽습니다.
- 컨트롤 `2/5`: 파워 성향이 더 크지만 코스 조절도 크게 불안하지 않은 편입니다.
- 편안함 `3/5`: 타구감이 비교적 부드럽고 팔 부담을 덜어주는 편입니다.

주의 문장 예시

- 강한 스윙을 하지 않으면 장점이 덜 살아날 수 있습니다.
- 타점이 늦으면 볼이 길어질 수 있습니다.

## 9. 스트링 추천 연결 규칙

라켓 단독 추천보다 "라켓 + 스트링 세팅" 추천이 훨씬 설득력 있다.

### 추천 원칙

- 라켓의 강점을 더 키우거나
- 약점을 보완하거나
- 사용자 성향에 맞게 세팅을 바꿔준다

### 스트링 추천 유형

- 컨트롤형 폴리
- 스핀형 폴리
- 소프트 폴리
- 멀티필라멘트
- 하이브리드

### 규칙 예시

- 파워 높음 + 컨트롤 낮음
  - 추천: 컨트롤형 폴리 또는 하이브리드
- 편안함 낮음 + 팔 부담 우려
  - 추천: 멀티필라멘트 또는 소프트 폴리
- 스핀 높음 + 중상급자
  - 추천: 스핀형 폴리
- 입문자 + 관용성 선호
  - 추천: 편안한 멀티 또는 부드러운 하이브리드

### 함께 보여줄 정보

- 추천 스트링 2~3개
- 추천 장력 범위
- 누구에게 맞는 조합인지
- 이 세팅을 권하는 이유

## 10. 그립 추천 연결 규칙

그립은 체감 만족도에 큰 영향을 준다.

### 추천 축

- 쿠션감
- 땀 대응
- 접지력
- 두께감
- 손에 닿는 감각

### 규칙 예시

- 편안함 보완이 필요함
  - 추천: 쿠션감 높은 리플레이스먼트 그립 + 부드러운 오버그립
- 여름철 실외 플레이 + 손땀 많음
  - 추천: 드라이 타입 오버그립
- 타구감 선명도를 선호
  - 추천: 얇은 오버그립

## 11. 액세서리 추천 구조

라켓 상세와 추천 결과 페이지에서는 아래 3단 구성으로 노출하는 것이 좋다.

1. 이 라켓과 잘 맞는 스트링
2. 이 라켓의 성향을 보완하는 그립
3. 선택형: 댐프너/가방/볼

이렇게 하면 장비 플랫폼으로서의 깊이가 생기고 객단가도 자연스럽게 올라간다.

## 15. 피해야 할 해석 방식

### 1. 서로 겹치는 항목을 따로 노출하는 방식

- 예: 같은 파워 개념을 서로 다른 이름으로 나누는 것
- 예: 같은 편안함 개념을 서로 다른 이름으로 나누는 것

이런 방식은 사용자가 차이를 이해하기 어렵고, 점수 해석이 흐려진다.

### 2. 플레이어 능력과 라켓 성향을 섞는 방식

- 예: "상급자용 점수", "복식력", "공격력" 같은 모호한 항목

이런 항목은 라켓 자체의 성능이 아니라 사용자 맥락이 섞여 객관성이 떨어진다.

### 3. 근거 없는 정밀도를 주는 방식

- 예: `컨트롤 87점`, `스핀 73점`을 그대로 공개하는 것

내부 정밀 점수는 유지하되, 사용자에게는 축 `N/5`와 총점 `N/15`의 정수 해석 점수로 단순화한다.

### 4. 브랜드 마케팅 문구를 재포장하는 방식

- 예: "폭발적 파워", "투어급 정밀성" 같은 카피성 문장

이 방식은 객관적 해석이 아니라 광고처럼 보인다.

### 5. 스펙 누락 상태에서 단정하는 방식

- 스윙웨이트, RA, 헤드사이즈 같은 핵심 값이 비어 있으면 신뢰도를 낮추거나 노출을 보수적으로 해야 한다.

## 16. 권장 결론

사용자 노출은 5축으로 가는 것이 맞다.

다만 아래 구조를 권장한다.

- 사용자 화면: 파워 / 컨트롤 / 스핀 / 편안함 / 안정성
- 내부 추천 엔진: 5축 + 보조 신호(조작성, 관용성, 팔 부담 리스크 등)

즉, 보여주는 것은 단순하게, 계산은 충분히 정교하게 가져가는 것이 가장 좋다.
## 17. 결론

이 플랫폼의 핵심은 "라켓을 읽는 새로운 한국어"를 만드는 것이다.

사용자는 결국 숫자보다 확신을 사고, 확신은 아래 조합에서 나온다.

- 신뢰되는 해석 점수
- 깨끗한 고화질 이미지
- 내 스타일과 연결된 설명
- 바로 이어지는 스트링/그립 추천

이 네 가지가 합쳐질 때 센세이셔널한 장비 플랫폼 포지션이 가능하다.
